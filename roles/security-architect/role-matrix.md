# ✅ Role-to-Deliverable Matrix

| Role                | Primary Deliverables                          | Tone                | Compliance Focus                |
|---------------------|----------------------------------------------|---------------------|---------------------------------|
| Security Architect  | Architecture diagrams, risk models          | Strategic, standards-driven | SOC 2, HIPAA, HITRUST          |
| Security Engineer   | SOPs, automation scripts, hardening guides  | Technical, precise  | CIS IG1–IG3, NIST CSF          |
| Network Security    | Segmentation checklists, firewall policies  | Technical, risk-aware| CIS IG2, NIST PR.AC            |
| IAM Security        | Identity governance templates, MFA rollout  | Technical, precise  | SOC 2 CC6.1, HIPAA §164.312    |
| DevSecOps           | CI/CD security templates, container hardening| Automation-first    | CIS Benchmarks, NIST CSF       |
| Security Operator   | Incident response runbooks, alert tuning    | Practical, detail-oriented| SOC 2 CC7.4, HIPAA §164.308    |
| Technical Writer    | SOP templates, KBAs, governance docs        | Clear, audit-ready  | SOC 2, HIPAA                   |
| Project Manager     | Roadmaps, JIRA Epics, status reports        | Organized, collaborative| SOC 2, HITRUST                |
| Manager             | Executive summaries, compliance reports     | Professional, persuasive| SOC 2, HIPAA, HITRUST         |

### ✅ Key Principle:
> Use this matrix to select the correct template and tone for each role.
